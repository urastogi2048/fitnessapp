# Contributing to FitnessDude

First of all, thank you for considering contributing to FitnessDude! 🎉

Whether you're fixing a bug, improving documentation, suggesting features, or writing code, your contributions are greatly appreciated.

## How to Contribute

### 1. Fork the Repository

Create your own fork of the repository and clone it locally.

```bash
git clone https://github.com/urastogi2048/fitnessapp.git
```

### 2. Create a Branch

Create a new branch for your changes.

```bash
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes

* Follow the existing code style.
* Keep commits focused and meaningful.
* Test your changes before submitting.

### 4. Commit Changes

Use clear commit messages.

Examples:

```text
feat: add workout streak tracking
fix: resolve profile image loading issue
docs: update installation guide
```

### 5. Push Your Branch

```bash
git push origin feature/your-feature-name
```

### 6. Open a Pull Request

Submit a Pull Request with:

* A clear title
* Description of the changes
* Screenshots (if UI changes are involved)
* Any relevant issue references

## Reporting Bugs

When reporting bugs, please include:

* Device information
* Operating system version
* Steps to reproduce
* Expected behavior
* Actual behavior
* Screenshots (if applicable)

## Feature Requests

Feature requests are welcome. Please describe:

* The problem you're trying to solve
* Your proposed solution
* Any alternatives considered

## Code Style

* Follow Flutter and Dart best practices.
* Keep code readable and well-structured.
* Use meaningful variable and function names.
* Add comments where necessary.

## Questions

If you have any questions, feel free to open an issue or start a discussion.

Thank you for helping improve FitnessDude! 💪
